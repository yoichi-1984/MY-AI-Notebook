# CHANGELOG (変更履歴)

本プロジェクトのバージョンごとの変更履歴です。各リリースは Git コミットハッシュおよび変更内容に整合しています。

## [0.3.0] - 2026-06-28
### 追加・改善
- **セマンティック検索の親子チャンク（Small-to-Big Retrieval）構成を導入**:
  - Azure Embedding API の上限（8,192トークン）に起因する「長文・大容量PDFのベクトル登録失敗」エラーを根本的に解消しました。
  - テキストを約2,000文字（200文字オーバーラップ）のチャンクに自動分割し、チャンク単位でベクトルを登録するアーキテクチャに移行しました。検索時は全チャンクでマッチングを行い、最もスコアの高かった「親ドキュメント（ページ/添付ファイル）」をフロントエンドへ返すため、UI側の変更は不要です。
  - LanceDB のテーブルを `knowledge_vector_table_v3` に更新し、スキーマに `source_id`（親ID）と `chunk_index`（分割番号）を追加しました。
  - 起動時に旧テーブルを検出した場合、既存の SQLite データから全ノート・添付ファイルを自動で再インデックス（チャンキング付き）します。データが消えることはありません。

### 修正
- **ベクトル登録処理の移行漏れを全て解消**:
  - `recalculate_vector_on_edit`（テキスト編集時）、`update_rules_with_feedback`（手動仕分け学習後）、`process_new_image_workflow`（画像追加後）、`process_attachment_ai_workflow`（添付ファイル解析後）のすべてで、旧関数（`upsert_vector_data`）の呼び出しが残っていた問題を修正しました。
  - `process_attachment_ai_workflow` 内で `target_page` 変数が未定義になる `NameError` を修正しました。
- **`delete_vector_data` のエラーログ `NameError` を修正**: 削除失敗時の例外メッセージで存在しない変数 `page_id` を参照していたバグを修正しました（`source_id` に修正）。
- **空文字列のEmbeddingリクエスト対策**: ノートのテキストが空の場合に空文字列 `""` をAzure APIに送信しないよう、スペース `" "` にフォールバックするガードを追加しました。
- **廃止済み `migrate_to_v5` 関数を安全化**: 現環境（DB v10）では絶対に呼ばれない旧マイグレーション関数が誤って呼び出された場合でも、v3テーブルを破壊しないよう処理を無効化（即 `return`）しました。

  - データベース接続時に `PRAGMA foreign_keys = ON` を有効化し、親不在のタスクや画像が残らないようデータ整合性を強化しました。あわせて、アプリ起動時に既存の孤立データを整理する自己修復処理を追加しました。

## [0.2.6] - 2026-06-28

## [0.2.5] - 2026-06-27
### 修正
- **各種非同期ワークフローにおけるオフライン時（API通信エラー時）のキューイング漏れと不整合の修正**:
  - **添付ファイル解析**: 要約生成やベクトル登録の内部 `try-except` 句で例外が握りつぶされていた問題を修正し、`OfflineException` を上位へ伝播させ、確実にジョブキュー (`api_jobs`) へ `"attachment"` ジョブとして退避させるよう改修。

  - **ドキュメントインポート**: インポート処理でオフラインエラー時に `failed` とせず `offline_queued` に移行して `document_import` ジョブとしてキューイング。さらに `finally` ブロックで一時インポートファイルを削除せず残すよう制御を追加。

  - ノートの完全削除を伴うフォルダ削除機能において、関連テーブルのレコード一括削除処理を `sqlite_client.py` 側から削除。完全削除ロジックは物理ファイル削除を担う `main.py` 側で既に実行されているため重複を排除。

## [0.1.9] - 2026-06-24
### 修正
- **未保存警告モーダル表示とノート切り替えフリーズバグの修正**:
  - ノートタイトル等の変更後に保存せずに他のノートに切り替えようとすると、画面が一切反応しなくなりフリーズする重大なバグを解決。
  - **`renderNotes()` の NullPointerError ガード**: 過去のデータに本文（`raw_text`）が `null` のノートが含まれる場合、`substring` を呼び出そうとしてクラッシュする不具合を防ぐため、安全な Null ガード `(note.raw_text || '')` を追加。
  - **`checkUnsavedChanges()` の Lucide エラーハンドリング**: オフライン環境等で Lucide スクリプトの読み込みが失敗している際に `lucide.createIcons()` の呼び出しで発生するクラッシュを防止するため、存在チェックと `try-catch` による保護を実装。
  - **`selectNote()` の堅牢化**: 警告チェック Promise の実行中に例外が発生しても、他のノートを開く操作をブロックしないように `try-catch` で保護し、安全な切り替え続行を保証。

## [0.1.8] - 2026-06-23
### 追加
- **512次元マルチベクトルRAG検索アーキテクチャ**:
  - これまで「要約＋タグ」の1つのベクトルのみだった検索対象を、「本文」「要約」「タグ」の3つの独立した512次元ベクトルへ分離し、それぞれに対して検索をかけるマルチベクトル構成へ移行。
  - ベクトルデータベース（LanceDB）のスキーマを `knowledge_vector_table_v2` に更新し、3種類のベクトルを格納するカラム（`summary_vector`, `tags_vector`, `body_vector`）を追加。
  - 全文検索（FTS）のインデックス対象を、手書き本文 `raw_text` と画像OCRテキスト `ocr_text` を結合した新カラム `fts_text` に変更し、手書きメモもキーワード検索でヒットするように拡張。
  - ベクトル検索3系統とFTS全文検索の計4つの結果順位を、RRF（Reciprocal Rank Fusion）で統合して並び替えるハイブリッドRRF融合検索を実装。
- **画像並行解析時の要約上書き競合防止（楽観的ロック）**:
  - `note_pages` テーブルに `analyzed_image_count`（要約作成時のOCR完了画像数）カラムを追加するマイグレーション（スキーマv8）を実装。
  - `update_page_metadata_merge` において、タスクの保持する画像数（世代スタンプ）がDBの現在値より小さい場合は、より多くの情報を含んだ要約がすでに存在する（他の処理が先に終わった）と判断して上書きをスキップする競合防止ガードを導入。
- **データ自動再インデックス・マイグレーション機能**:
  - 起動時に古い1536次元テーブルから新512次元マルチベクトルテーブルへ、SQLiteの全既存データをバックグラウンドで自動再計算・移行するインポートタスクを追加。

### 変更
- **APIコールの非同期並行化（処理時間短縮）**:
  - ノート保存や画像解析などのバックグラウンド処理において、`asyncio.gather` を用いて「本文」「要約」「タグ」のEmbedding API（Azure OpenAI）リクエストを完全並行送信化。ネットワーク往復の待ち時間を直列時の約1/3に削減。
- **手動編集時のインデックスバグ修正**:
  - 通常フォルダのノートをユーザーが手動編集した際、古い要約でベクトルが再計算されて本文の変更が反映されない不具合を解消。編集時にも常に最新の本文 `raw_text` を取得して `body_vector` および `fts_text` が更新されるよう改善。
- `/api/search` エンドポイントを `async def` に変更し、クエリの512次元ベクトル生成を `await` で待機するように対応。

### 修正
- **並行処理時の `asyncio` 未定義エラーの解消**:
  - `services/workflow.py` において `import asyncio` がグローバルスコープでインポートされておらず、複数の画像解析や削除が連続して非同期で走った際に `NameError: name 'asyncio' is not defined` が発生するバグを修復。
- **未保存警告チェック時のJSクラッシュ（画面遷移不能バグ）の修正**:
  - 過去に作成されたノートでタグ（`ai_tags`）が `null` である場合に、`hasUnsavedChangesForCurrentPage()` 関数内で `null.split(',')` が実行されJSエラーでクラッシュし、他のノートへの切り替えや警告ポップアップの表示が一切行えなくなる不具合を解消。
  - `pageTags` に対する Null ガードを追加し、関数全体を `try-catch` で囲むことで安全な防御的制御へリファクタリング。

## [0.0.1] (Initial commit) - 2026-06-20
### 追加
- **初版リリース**: FastAPI (バックエンド) + SQLite (階層/メタデータ) + LanceDB (ベクトルDB) + Vanilla JS/Tailwind (フロント) による完全ローカル運用のナレッジデータベースの構築。
